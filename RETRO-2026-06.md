# Retrospective — portfolio improvement arc (June 2026)

Summary of four slice retrospectives plus integration learnings. **Phase 7** in [IMPROVEMENT-PLAN.md](IMPROVEMENT-PLAN.md) tracks fixes applied from this doc.

## What went well

- Phased IMPROVEMENT-PLAN gave durable IDs across sessions and subagents.
- HTML-first content + `<template>` GitHub shells reduced XSS surface and simplified tests.
- sessionStorage cache + load-generation guard made GitHub resilient without a build step.
- Connect IA simplification (proof before ask, quiet closing section) matched user intent.

## Issues found → fixes

| Issue                                             | Impact                                             | Fix (Phase 7)                                                        |
| ------------------------------------------------- | -------------------------------------------------- | -------------------------------------------------------------------- |
| Double random theme (inline head + `initTheme()`) | Flash / mismatch after FOUC guard                  | `theme-bootstrap.js` + `initTheme()` reads existing `data-theme`     |
| README claimed `localStorage` persistence         | Wrong mental model for users/agents                | Session-only switcher; stale key cleared on init                     |
| Duplicated theme pool logic                       | Drift risk                                         | `theme-pick.js` shared by tests + `theme.js`; bootstrap mirrors pool |
| Tab-only cursor keyboard mode                     | Mouse users clicking focusables kept hidden cursor | `focusin` on focusables sets `using-keyboard`                        |
| Smoke-only CI                                     | Scroll/focus regressions undetected                | Playwright E2E + axe in `npm run test:all`                           |
| Untested load generation                          | Race bugs could return                             | `beginGitHubLoad` / `isGitHubLoadCurrent` + unit test                |
| GitHub skeleton `aria-label` on bare `div`        | axe CI failure                                     | `role="status"` on skeleton template                                 |
| Stale IMPROVEMENT-PLAN “Top 5”                    | Mis-prioritized next work                          | Phase 7 table + workflow docs                                        |
| Subagent integration gaps                         | Focus steal, plan drift                            | [docs/IMPROVEMENT-WORKFLOW.md](docs/IMPROVEMENT-WORKFLOW.md)         |

## Process rules (keep)

1. **One plan ID per PR/subagent** — parent owns seams (boot order, focus, test counts).
2. **Post-merge checklist** — `npm run test:all`, load at scroll 0, tab grid, update plan.
3. **Map contact surfaces** before IA changes — see [docs/UX-PRINCIPLES.md](docs/UX-PRINCIPLES.md).
4. **Never `.focus()` on load** for decorative or async widgets.
